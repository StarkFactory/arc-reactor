# 고사양 개발 장비 도입 필요성

## 현재 작업 방식

러닝메이커솔루션팀에서 2개의 프로젝트 (러닝메이커, 커뮤니티)를 진행하고 있으며 필드앤포럼 아슬란에서는 8개의 프로젝트를 진행중입니다. 소수의 인원으로 병행하고 있으며, Claude Code 2개 + Codex 1개, 총 3개의 AI 에이전트를 동시에 노트북에서 구동하여 병렬 개발을 진행하고 있습니다. 서브 에이전트 기능들을 활용해서 에이전트 내부에서도 여러개의 에이전트를 구동하여 한쪽에서 백엔드 API를 구현하는 동안, 다른 쪽에서 기획과 보안 스캔을, 나머지 하나에서 프론트엔드 작업과 테스트 코드 작성을 동시에 수행하는 방식입니다.

여기에 아슬란에서 진행중인 리액터(AI Agent)는 사내 시스템으로 도입할 예정이며 이 에이전트 구동만으로도 Spring Boot JVM 서버, PostgreSQL, Redis, MCP 서버 3~7개, Vite 개발 서버, IDE, 브라우저까지 동시 구동하며 검증작업을 진행해야하므로 현재 환경에서는 메모리 스왑이 빈번하게 발생합니다. 에이전트 2개만 돌려도 빌드 시간이 2~3배 늘어나고 타임아웃이 발생하여, 3개를 온전히 활용하지 못하고 있습니다.

## 병렬 에이전트 활용의 실제 생산성 (클로드 코드 활용 Git 커밋 기준)

최근 1일 (04-07) — 6개 프로젝트, 107커밋, 466파일 변경, +18,783줄
- 러닝메이커: 채점 신규기능 개발, 기능 QA
- 커뮤니티: 정책 문서 코드 반영, 기능 QA
- Reactor: Admin API 8건 일괄 구현, 리팩토링 4건, 테스트 추가
- Atlassian MCP: 보안 취약점 7개 카테고리 전면 수정, Rate Limiter 구현
- Paravel App: 디자인 시스템 11종 구현 + 기존 화면 전면 전환
- Pinggo: 미스터리 박스 당첨자 시스템, 300명 동시접속 최적화

최근 1주 (03-31 ~ 04-07) — 10개 프로젝트, 562커밋, 3,940파일 변경, +159,424줄
- 러닝메이커: 채점 신규기능 개발, QA 작업
- 커뮤니티: 정책 문서 코드 반영, 코드 리팩토링, QA 작업
- Paravel Backend 232커밋: DDD 전면 리팩토링, E2E 테스트 26→74개
- Paravel App 148커밋: 디자인 시스템 도입, QA 사이클 26회
- Arc Reactor 96커밋: 멀티에이전트 Phase 1, Admin API 15개 완성
- 그 외 Clipping MCP, Pinggo, Arc Reactor Web/Admin, Atlassian MCP

AI 도구 없이 수작업 기준 최소 4~6주 분량입니다. 단순 코드 생성이 아니라 아키텍처 설계, 보안 스캔, E2E 테스트, 부하 테스트, 문서 자동 생성까지 품질 작업을 병행하고 있습니다.

## 로컬 LLM 실행 계획

Reactor 고도화를 위해 Ollama 기반 로컬 LLM 실행, 임베딩 모델 운용, 파인튜닝 모델 테스트를 계획하고 있으며, GPU 가속과 충분한 메모리가 필수적입니다.

## 결론

64GB 고사양 장비는 AI 에이전트 3개 병렬 구동 + 개발 서버 다중 실행 + 로컬 LLM 테스트를 동시에 수행하기 위한 필수 기반입니다. 1주 562커밋, 159,424줄의 생산성은 AI 도구의 원활한 동시 구동에 직접 의존하며, 현재 병목을 해소해야 10개 프로젝트 병행 속도를 유지할 수 있습니다.
