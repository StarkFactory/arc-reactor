# v3.9.2 릴리즈 노트

일자: 2026-02-18

## 주요 변경

- 태그 기반 GitHub Release 자동화를 추가했습니다.
  - 이제 `v*` 태그를 푸시하면 CI가 릴리즈를 자동 생성합니다.
- CI 테스트 시간 예산 가드를 추가했습니다.
  - 기본(unit) 스위트: 90초
  - 통합 API 스위트: 150초
- MCP 매니저를 책임 단위로 분리해 유지보수성을 높였습니다.
  - 연결 지원, 재연결 조정, 스토어 동기화를 분리했습니다.
- MCP 콜백 실패 경로에서 코루틴 취소 전파 semantics를 보강했습니다.

## 운영 영향

- 릴리즈 생성 과정이 더 일관되고 수동 작업이 줄어듭니다.
- 테스트 시간 회귀를 CI에서 조기에 감지할 수 있습니다.
- MCP 런타임 코드를 확장/리뷰하기 쉬워졌습니다.

## 업그레이드 메모

- 프로젝트 버전은 `3.9.2`입니다.
- 기존 `v3.9.1` 릴리즈는 그대로 유지됩니다.
- 신규 태그부터 `.github/workflows/release.yml`이 자동 적용됩니다.
