**버그의 실체 (scanner 분석 확장)**:

1. `ExecutionResultFinalizer.mergeVerifiedSourcesMetadata` (기존 line 663):
   ```kotlin
   metadata["grounded"] = latestSignal?.grounded ?: verifiedSources.isNotEmpty()
   ```
2. Kotlin의 Elvis `?:` 연산은 **좌변이 null일 때만** 우변을 평가한다. `latestSignal?.grounded`가 `false`(non-null)이면 우변 `verifiedSources.isNotEmpty()`는 완전히 무시된다.
3. `ToolResponseSignalExtractor`는 도구 JSON에 `grounded: false`가 명시적으로 있으면 `Boolean.FALSE`를 세팅한다. Partial match, 불완전 조회, 일부 도구의 conservative 기본값 등이 이 경로를 트리거.
4. 동시에 `ToolCallOrchestrator.collectParallelResults` / `invokeAndFinalizeDirect`는 `VerifiedSource` 목록을 `HookContext.verifiedSources`에 누적. URL 추출이 정상적으로 동작했다면 근거 목록은 여전히 존재.
5. 따라서 tool이 "part of answer is not fully grounded" 의미로 `grounded=false`를 보내도, 실제 source URL은 여전히 누적되고 응답에도 인용 링크가 포함된다. 그러나 최종 metadata에는 `grounded=false`가 기록되어 모순 발생.

#### Fix

`mergeVerifiedSourcesMetadata`의 한 줄만 변경:

```kotlin
// BEFORE
metadata["grounded"] = latestSignal?.grounded ?: verifiedSources.isNotEmpty()

// AFTER (R342)
metadata["grounded"] = (latestSignal?.grounded == true) || verifiedSources.isNotEmpty()
```

**정책 명확화**:
- **signal=true** → 그대로 `grounded=true` (tool이 긍정적 신호를 보냈으므로 존중)
- **signal=false + sources.isNotEmpty()** → `grounded=true` (실제 근거가 존재하므로 tool의 false signal보다 우선)
- **signal=false + sources 없음** → `grounded=false` ("근거 없음 + tool이 이를 명시적 확인"의 정확한 의미)
- **signal=null + sources.isNotEmpty()** → `grounded=true` (기존 동작 유지)
- **signal=null + sources 없음** → `grounded=false` (기존 동작 유지)

이 수정은 "근거가 있으면 grounded"라는 직관적 도메인 규칙을 복원한다. Tool signal의 `false`는 "추가 문맥 정보"로 취급되며 실제 verifiedSources 존재가 우선한다.

#### 설계 메모

- **대안 1**: `latestSignal?.grounded == false && verifiedSources.isEmpty()`로 "둘 다 없음"만 false 처리. 채택 안 함 — `(signal == true) || sources.isNotEmpty()`가 boolean 의미상 더 명료하고 null 처리도 동일.
- **대안 2**: `signal=false`를 완전히 무시하고 `verifiedSources.isNotEmpty()`만 보기. 채택 안 함 — tool이 "근거 없음"을 명시적으로 확인하는 경로를 유지해 sources 없을 때의 정확도 보존.
- **대안 3**: `ToolResponseSignal.grounded` 필드 자체를 `Boolean?` → 3-state enum으로 확장. 채택 안 함 — breaking change, 현재 문제는 boolean OR로 해결.
- **채택**: 1줄 boolean OR. 최소 변경 + 도메인 의미 명확화.
- §0.1 MCP 계약: 완전 무관. tool signal은 MCP wire protocol과 독립된 Arc Reactor 내부 추출 레이어.
- §0.2 Redis cache: 완전 무관.
- §0.3 대화 컨텍스트: 완전 무관. metadata는 response-scoped.
