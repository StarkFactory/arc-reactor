#### Fix

**1. `distinctBy` key selector 변경** (`VerifiedSource.kt:43`):

```kotlin
// BEFORE
.distinctBy { it.url }

// AFTER (R343)
.distinctBy { normalizeUrlForDedup(it.url) }
```

**2. `normalizeUrlForDedup` private helper 추가**:

```kotlin
private fun normalizeUrlForDedup(url: String): String {
    val trimmed = url.trim()
    val noFragment = trimmed.substringBefore('#')
    if (noFragment.length <= 1) return noFragment
    return if (noFragment.endsWith('/') && !noFragment.endsWith("://")) {
        noFragment.trimEnd('/')
    } else {
        noFragment
    }
}
```

**정규화 정책**:
1. `#fragment` 제거 — HTML anchor는 같은 문서 내 위치 표식이므로 dedup 대상
2. trailing slash 정리 — `/path/` → `/path` (path 의미 변화 없음)
3. `scheme://` 엣지 케이스 보존 — `https://` 같은 불완전 URL은 그대로 두어 잘못된 정규화 방지
4. 길이 ≤ 1 엣지 케이스 보존 (`/` 단일 문자 등)
5. **대소문자는 변경 안 함** — path는 대소문자 구분, raw 값 보존이 디버깅에 유리
6. **query string은 변경 안 함** — query 파라미터가 다르면 실제로 다른 자원일 수 있음 (`?id=1` vs `?id=2`)

**원본 URL 보존**: `distinctBy`는 key selector를 사용해 dedup하지만 원본 요소(= `VerifiedSource`)는 그대로 유지한다. 응답에 표시되는 URL은 **가장 먼저 발견된 변형**이 그대로 살아남아 사용자에게 자연스러운 형식(trailing slash 유지 등)으로 보여진다.

#### 설계 메모

- **대안 1**: `URI` 파싱 + `URI.normalize()` 사용. 채택 안 함 — `URI` 생성자가 잘못된 URL(MCP 도구가 가끔 반환하는 비표준 형식)에 대해 예외를 던질 수 있고, 예외 처리를 추가하면 복잡도 증가. 간단한 문자열 정규화로 충분.
- **대안 2**: query string도 정규화 (sort parameters). 채택 안 함 — query 순서 차이는 실제로 같은 자원이 아닐 수도 있어 over-normalization 위험.
- **대안 3**: host 소문자화. 채택 안 함 — 현업 Confluence/Jira URL은 host가 항상 같은 형식으로 오므로 불필요한 변형.
- **대안 4**: `data class VerifiedSource`에 `canonicalUrl` 필드 추가. 채택 안 함 — public API 변경, breaking.
- **채택**: `distinctBy` key selector로 정규화된 키만 사용, 데이터 모델 불변. 최소 변경 + 명확한 효과.
