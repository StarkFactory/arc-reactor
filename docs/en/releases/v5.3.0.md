# v5.3.0

Release date: 2026-03-16

## Highlights

- **API consistency and error handling**:
  - `OUTPUT_GUARD_REJECTED` and `OUTPUT_TOO_SHORT` error codes now return HTTP 422 instead of 500, correctly classifying them as validation errors (#327).
  - Error responses standardized with shared helper methods across controllers (#331).

- **Code quality and refactoring**:
  - `DynamicSchedulerService` helpers extracted to comply with method length limits (#328).
  - Intermediate list allocation removed in `findToolAdapter` for collection efficiency (#330).

- **Infrastructure and performance**:
  - `run_id` index added to `pending_approvals` table for faster approval lookups (#329).
  - Helm resource requests increased for production workloads (#326).

## Compatibility and Migration

- **HTTP status change**: `OUTPUT_GUARD_REJECTED` and `OUTPUT_TOO_SHORT` responses now return 422 (Unprocessable Entity) instead of 500. Clients checking for HTTP 500 on these error codes should update to expect 422.
- No breaking API path or configuration key changes from v5.2.0.

## Operational Notes

- Version bumped to `5.3.0`.
- Tag `v5.3.0` triggers GitHub Release and Docker publication workflows.
- Database migration: new index on `pending_approvals(run_id)` is applied automatically via Flyway when enabled.
