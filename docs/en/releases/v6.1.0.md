# v6.1.0 — Production Hardening & Code Quality

**Release Date:** 2026-03-27
**Commits since v6.0.0:** 34

## Highlights

Post-v6.0.0 stabilization release focused on production hardening, code quality, and operational visibility. No new features — pure robustness improvements.

## Performance

- **JdbcMemoryStore**: N+1 batch delete — 1000 expired sessions now cleaned in 2 queries instead of 1001 (#535)
- **JdbcPendingApprovalStore**: Cleanup moved from every-request to 5-minute time-gated intervals (#538)
- **pending_approvals**: Added `resolved_at` index for cleanup query performance (#537)
- **ConversationMessageTrimmer**: Unbounded `ConcurrentHashMap` → bounded Caffeine cache (10K, 5min TTL) (#539)
- **SlackUserRateLimiter**: O(n*m) cleanup → 5-second time-gated (#541)

## Security

- **MCP adminUrl SSRF**: Added `SsrfUrlValidator` check at MCP server registration (#fcb584b8)
- **FeedbackController**: Added `@Valid` + `@Size` constraints on all input fields (#fcb584b8)
- **arc-admin @Valid**: Added missing `@Valid` on 5 endpoints (#536)
- **Exception message masking**: Removed `e.message` exposure in TenantAdminController, OutputGuardRuleController, ProactiveChannelController (#557)
- **Unsafe casts**: Converted 11 `as` → `as?` safe casts across MetricQueryService, SloService, StageTimingSupport, ToolCallOrchestrator, PromptRequestSpecBuilder, RetryExecutor (#555, #443ec502)
