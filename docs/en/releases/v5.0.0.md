# v5.0.0

Release date: 2026-03-14

## Highlights

- **Security hardening (deny-by-default)**:
  - Session ownership now defaults to deny-by-default, closing a memory leak for orphaned sessions (#283).
  - SSRF validation added for MCP server SSE URL registration, blocking internal-network targets (#288).
  - SnakeYAML parsing switched to `SafeConstructor` to prevent deserialization attacks (#273).
  - `@Size` validation annotations added to all request DTOs for input boundary enforcement (#280).
  - Admin audit trail records all sensitive operations with actor identity and timestamp (#287).
  - `system-prompt-max-chars` default set to 50,000 to prevent prompt-stuffing (#268).

- **Reliability and correctness**:
  - `JdbcMemoryStore.evictOldMessages` made atomic to prevent partial eviction under concurrent access (#272).
  - `DefaultCircuitBreaker` HALF_OPEN state transitions now use CAS to eliminate race conditions (#271).
  - Control characters in `TimescaleSpanExporter` JSON output are now properly escaped (#269).
  - Streaming retry path added for `Flux` errors, preventing silent failures on transient LLM errors (#277).

- **Performance and observability**:
  - N+1 query eliminated in `JdbcMemoryStore` session listing (#278).
  - Caffeine cache added to `DefaultTokenEstimator` for repeated token count lookups (#285).
  - Composite database indexes added for high-frequency query patterns (#281).
  - Database and MCP server health indicators registered for Spring Boot Actuator (#289).
  - `X-Request-ID` correlation filter propagates request IDs across logs and responses (#282).
  - SLA metrics added: LLM latency percentiles (P50/P95/P99), tool output size, active request gauge (#286).
  - JSON structured logging format for production log aggregation (#284).

- **Code quality and CI**:
  - 18 `Thread.sleep()` calls replaced with non-blocking coroutine alternatives in admin tests (#275).
  - Hardcoded tool names extracted from `ToolCallOrchestrator` to configuration (#276).
  - CI safety gate migrated from hardcoded test lists to JUnit tag-based filtering (#274).
  - Gradle `testLogging` configured to print failure details on test runs (#270).

## Compatibility and Migration

- **Session ownership** is now deny-by-default. Existing deployments that rely on anonymous session access should set `arc.reactor.security.session-ownership-enabled: false` to preserve prior behavior.
- **Input validation** annotations may reject previously accepted oversized request fields. Review `@Size` constraints on DTO fields if clients send large payloads.
- No breaking API path or configuration key changes from v4.8.0.

## Operational Notes

- Version bumped to `5.0.0`.
- Tag `v5.0.0` triggers GitHub Release and Docker publication workflows.
- New health endpoints: `/actuator/health/db` and `/actuator/health/mcp` available when Actuator is enabled.
- Structured JSON logging is opt-in via `arc.reactor.logging.format: json`.
