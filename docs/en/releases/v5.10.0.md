# v5.10.0

Release date: 2026-03-16

## Highlights

- **Rate limit HTTP status fix**:
  - `PreExecutionResolver` was mapping all guard rejections (including rate limiting) to `AgentErrorCode.GUARD_REJECTED`, which `ChatController` mapped to HTTP 403. Rate-limited requests now correctly use `AgentErrorCode.RATE_LIMITED`, producing HTTP 429 (Too Many Requests) as documented in the API spec (#355).

- **Kotlin defaults aligned with application.yml**:
  - Six Kotlin property defaults corrected to match `application.yml` values (#356).
  - Helm chart `temperature` corrected from 0.7 to 0.1 and `maxToolsPerRequest` from 20 to 30 (#356).

- **Documentation consistency**:
  - 11 documentation files updated for consistency with current code defaults (#356).

## Compatibility and Migration

- **HTTP 429 for rate limiting**: Rate-limited requests now return HTTP 429 instead of 403. Clients that retry on 403 should also handle 429 with appropriate backoff.
- No breaking API path or configuration key changes from v5.9.0.

## Operational Notes

- Version bumped to `5.10.0`.
- Tag `v5.10.0` triggers GitHub Release and Docker publication workflows.
