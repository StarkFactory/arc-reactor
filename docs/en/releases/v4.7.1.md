# v4.7.1

Release date: 2026-03-01

## Highlights

- **Slack runtime hardening**:
  - Fixed metrics recorder auto-configuration so Micrometer metrics are emitted when `MeterRegistry` is available.
  - Added tracked-thread gating to avoid replying to unrelated Slack thread messages.
  - Improved event-path delivery observability by logging non-`ok` Slack send outcomes.
- **Error report stability**:
  - Enforced `arc.reactor.error-report.request-timeout-ms` with coroutine timeout handling in async processing.
- **Validation reliability**:
  - Updated `scripts/dev/validate-slack-runtime.sh` to detect transport mode (`events_api` vs `socket_mode`) and skip unsupported checks.

## Compatibility and Migration

- No API path changes.
- No required configuration migrations.
- Optional new Slack thread-tracking properties are enabled by default:
  - `arc.reactor.slack.thread-tracking-enabled`
  - `arc.reactor.slack.thread-tracking-ttl-seconds`
  - `arc.reactor.slack.thread-tracking-max-entries`
