# v5.1.0

Release date: 2026-03-16

## Highlights

- **RAG pipeline enhancements** (academic-grounded):
  - Adaptive Query Router classifies queries into no-retrieval, single-step, or multi-step paths based on complexity (Jeong et al., 2024 — Adaptive-RAG) (#303).
  - Query Decomposition splits multi-hop questions into independent sub-queries for parallel retrieval (Zhou et al., 2023) (#301).
  - LLM-based Contextual Compression removes irrelevant passages from retrieved documents before generation (Xu et al., 2024) (#300).
  - Parent Document Retriever expands small matched chunks to their full parent document for richer context (Chen et al., 2024) (#302).
  - RAG retrieval outcomes differentiated with log levels and Micrometer metrics for observability (#296).
  - Academic references documented in `docs/en/reference/rag-papers.md` (#299).

- **Multi-agent improvements**:
  - Per-node timeout support in `AgentNode` prevents individual agents from blocking orchestration (#312).
  - Per-node token usage tracking enables cost attribution across multi-agent graphs (#305).
  - Structured error propagation surfaces failed node details to the supervisor with error context (#294).

- **Scheduler enhancements**:
  - Comprehensive input validation added to `validateSchedule` (#293).
  - Execution history retention policy configurable per job (#306).
  - Template variable resolution (parameterized prompts) for scheduled job inputs (#309).
  - Tag-based job filtering for organized job management (#307).

- **Agent reliability fixes**:
  - Message pair integrity enforced in `ManualReActLoopExecutor` — `AssistantMessage` and `ToolResponseMessage` are always added/removed together (#297).
  - Guard stage order validation rejects duplicate `order` values in `GuardPipeline` (#292).
  - Runtime debug logs removed from repository (#291).

- **API and developer experience**:
  - Pagination support (`page`, `size`, `sort`) added to all list API endpoints (#298).
  - Tool result caching with Caffeine deduplicates identical tool calls within a single request (#310).
  - User memory prompt injection toggle (`inject-into-prompt`) controls whether memory is appended to system prompts (#308).
  - Citation auto-formatting appends verified source references to agent responses (#311).
  - HookExecutor KDoc and ConversationManager logging improved for maintainability (#295).

## Compatibility and Migration

- **Pagination**: List endpoints now return paginated responses by default (page 0, size 20). Clients consuming full lists should pass `size=1000` or iterate pages.
- **Tool result caching** is enabled by default for idempotent tool calls. Disable with `arc.reactor.features.tool-result-caching-enabled: false` if tools have side effects.
- No breaking API path or configuration key changes from v5.0.0.
