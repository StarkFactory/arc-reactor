# v5.8.0

Release date: 2026-03-16

## Highlights

- **Concurrency bug fixes**:
  - `CircuitBreaker` CLOSED-to-OPEN transition uses `compareAndSet()` instead of `set()`, preventing duplicate transitions and metrics under concurrent failures (#346).
  - `InMemoryRagIngestionPolicyStore.delete()` uses `getAndSet()` for atomic delete, preventing race conditions (#346).
  - `SlackSocketModeGateway.start()` uses `AtomicBoolean` CAS instead of a volatile flag, preventing double-start under concurrent `start()` calls (#346).
  - `AgentTracingHooks` no longer uses direct `MDC.put/remove` (unreliable in suspend functions); traceId and spanId are stored in `hookContext.metadata` instead (#346).

- **Error handling improvements**:
  - `IllegalArgumentException` handler added to `GlobalExceptionHandler`, returning HTTP 400 (#347).
  - `IllegalStateException` handler added to `GlobalExceptionHandler`, returning HTTP 500 (#347).

- **ChatOptions expansion**:
  - `topP`, `frequencyPenalty`, and `presencePenalty` parameters added to `LlmProperties` (null defaults) (#347).
  - Sampling parameters wired through `ChatOptionsFactory` to all three builder paths (GoogleGenAi, ToolCallingChatOptions, plain ChatOptions) (#347).

## Compatibility and Migration

- **New LLM parameters**: `topP`, `frequencyPenalty`, and `presencePenalty` default to null (no change to existing behavior). Set via `arc.reactor.llm.top-p`, `arc.reactor.llm.frequency-penalty`, and `arc.reactor.llm.presence-penalty`.
- **MDC tracing**: If custom hooks read traceId/spanId from MDC, migrate to reading from `hookContext.metadata["traceId"]` and `hookContext.metadata["spanId"]`.
- No breaking API path or configuration key changes from v5.7.0.
